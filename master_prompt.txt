================================================================================
CONTEXT — What you are building
================================================================================

Build a production-grade multi-agent HR Copilot that answers employee HR
questions using:
  - A proper multi-agent framework (BaseAgent, AgentRegistry, ParallelAgentExecutor)
  - Hybrid RAG retrieval (FAISS vector + BM25 keyword + RRF fusion)
  - Parallel specialist agent execution (fan-out pattern)
  - Cross-encoder reranking + NLI fact-checking
  - POSH/legal compliance gate with hard blocking
  - Local RAGAS evaluation (faithfulness, relevancy, context precision)
  - Streamlit web UI with 5 tabs
  - No external API keys required (runs fully locally)

Tech stack: Python 3.10+, sentence-transformers, faiss-cpu, rank-bm25,
transformers, pandas, streamlit, colorama, python-dotenv

The system is built for a workshop to teach students multi-agent
AI system design. Every file must have clear educational comments explaining
the WHY behind each design decision.

3.2 Create component_c_policy_data_agents.py:

    IMPORTS: from agent_framework import BaseAgent

    CONSTANTS:
    TOP_K_VEC = 9, TOP_K_BM25 = 7, TOP_K_RRF = 6, RRF_K = 60

    class PolicyRAGAgent(BaseAgent):
        @property
        def agent_name(self) -> AgentName: return AgentName.POLICY_RAG

        _vector_search(query, category_filter, top_k=TOP_K_VEC) -> List[Tuple[int, float]]
            FAISS inner product search with TWO-PASS category filter:
            Pass 1: collect chunks from priority category
            Pass 2: backfill from other categories if needed
            Return merged[:top_k]

        _bm25_search(query, category_filter, top_k=TOP_K_BM25) -> List[Tuple[int, float]]
            BM25 search with same two-pass category filter

        _rrf_fusion(vec_results, bm25_results, k=RRF_K) -> List[Tuple[int, float]]
            RRF formula: score[idx] += 1.0 / (k + rank)
            Apply to both vec_results and bm25_results
            Return sorted by score descending

        _infer_category_for_subquery(sub_question) -> Optional[List[str]]
            Map ESOP→compensation, leave→leave, notice→compensation,
            remote→remote_work, onboard→onboarding, posh→grievance,
            learning→learning. Return None if no match.

        retrieve(sub_query, plan, top_k=TOP_K_RRF) -> List[RetrievedChunk]
            For MULTI_DOMAIN: use _infer_category_for_subquery
            Otherwise: use plan.priority_docs
            Run _vector_search + _bm25_search + _rrf_fusion
            Return List[RetrievedChunk] with all scores populated

        run(plan) -> AgentResponse
            For each sub_query in plan.sub_queries: call retrieve()
            Deduplicate by chunk_id (boost rrf_score for cross-query hits)
            Sort by rrf_score desc, keep top 8
            Format raw answer from top-4 chunks with source citations
            Return AgentResponse with sources, confidence=deduped[0].rrf_score

    class DataQueryAgent(BaseAgent):
        @property
        def agent_name(self) -> AgentName: return AgentName.DATA_QUERY

        DATA_DIR = "data/hr_structured"

        _load_salary_bands() -> dict
            Lazy load salary_bands.json

        _load_headcount() -> DataFrame
            Lazy load headcount.csv via pandas

        query_salary_bands(question) -> str
            Extract band number (B1-B6) or title keyword from question
            Return formatted band details. If no match: return full band table.

        query_headcount(question) -> str
            Department-specific lookup, company-wide summary, attrition sort
            Return formatted table with source citation

        run(plan) -> AgentResponse
            Route sub_queries to query_salary_bands or query_headcount
            Detect keywords: "salary","ctc","band" → salary_bands
                             "headcount","attrition","employees" → headcount
            Return AgentResponse(data_used=True, confidence=0.92)

        # Input: (question, chunks, agent_response)
        # Output: (verified_chunks, ComplianceCheckResult)

        def __init__(self):
            self._reranker = None
            self._nli = None

        _load_reranker() -> CrossEncoder
            Lazy load RERANK_MODEL

        _load_nli() -> pipeline
            Lazy load NLI_MODEL using hf_pipeline("text-classification")

        rerank(question, chunks, threshold=RERANK_THRESH) -> List[RetrievedChunk]
            Create (question, chunk.text[:512]) pairs
            Score with cross-encoder
            Filter by threshold, sort by score desc, return top-8
            Set chunk.rerank_score for each

        fact_check(question, chunks) -> List[RetrievedChunk]
            For each chunk: nli_input = f"{chunk.text[:400]} [SEP] {question}"
            Parse NLI output: label "2" or "ENTAIL" → entailment score
            chunk.verified = True if entailment >= NLI_THRESH
            Return verified chunks. Fallback: return top-3 if none pass.

        compliance_scan(question, answer) -> ComplianceCheckResult
            Scan question + answer against SENSITIVE_PATTERNS
            Add caveats for each match
            SPECIAL RULE: if "posh" in flags AND question contains
            r"\b(file|lodge|register|report|raise)\b.*\b(complaint|harassment)\b"
            → passes=False, corrected_answer = ICC contact message

        run(question, chunks, agent_response) -> Tuple[List[RetrievedChunk], ComplianceCheckResult]
            reranked = rerank(question, chunks)
            verified = fact_check(question, reranked)
            raw_ans = agent_response.answer if agent_response else join chunk texts
            compliance = compliance_scan(question, raw_ans)
            return verified, compliance

    RAGAS EVALUATION FUNCTIONS:

    _cosine(a, b) -> float
        dot(a,b) / (norm(a) * norm(b))

    evaluate_faithfulness(answer, verified_chunks, nli_model) -> float
        Split answer into sentences (split on ". ")
        For each sentence: NLI check against context (top-4 chunks joined)
        entailment score ≥ 0.45 → counts as faithful
        Return faithful_count / total_sentences

    evaluate_relevancy(question, answer, embed_model) -> float
        Cosine similarity of question and answer embeddings
        Scale: max(0, min(1, (sim - 0.15) / 0.65))

    evaluate_context_precision(answer, verified_chunks) -> float
        For structured sources (salary_bands.json, headcount.csv in answer) → return 1.0
        For policy chunks: count chunks whose source_file or text 4-grams appear in answer
        Return used / total. Boost to 0.85 if structured source cited.

    evaluate_response(hr_response, verified_chunks, nli_model, embed_model) -> FinalHRResponse
        Compute all 3 RAGAS metrics, update hr_response fields, print results
